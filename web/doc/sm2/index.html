SM2 数字签名与公钥加密

SM2 是椭圆曲线公钥算法，可用于数字签名与验签、密钥协商（ECDH）、公钥加密 / 私钥解密。本工具在浏览器本地完成运算，不提供任何远程 HTTP 接口。

在在线工具里怎么用

打开 在线工具 → SM2 区块，按页面提示填写：用户标识（ID）、消息、随机数 k、公私钥等，点击对应按钮即可得到签名 (r,s)、密文或共享密钥。

本地 浏览器内计算 · 无服务端 URL

本地模块接口（gm.c / gm.js）

下表为 C 函数名；在页面里用 Module.cwrap('gm_xxx', …)（不要写前导下划线）。引入脚本 + 完整调用示例请去 开发者接口 → 复制使用。

函数 作用
gm_sm2_generate_keypair 由 32 字节熵生成私钥与未压缩公钥（64 字节 X‖Y）。
gm_sm2_sign / gm_sm2_verify 对消息签名、验签；需 ID 与长度参与 ZA 计算。
gm_sm2_encrypt / gm_sm2_decrypt 公钥加密、私钥解密；order_c1c3c2 控制密文分量顺序。
gm_sm2_ecdh_shared_key ECDH 派生 32 字节共享密钥。

输出数据形态

// 签名成功：r、s 各 32 字节，常以十六进制展示
{
  "r": "hex 64 字符",
  "s": "hex 64 字符"
}
// 加密结果：C1C3C2 顺序由实现与 order 参数约定，联调时请与对端一致
