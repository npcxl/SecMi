memory C 语言实现逻辑

  • 入口与导出：gm.c 作为 Emscripten 与本地共用的统一入口，通过 GM_KEEPALIVE 导出可供 JS/WASM 调用的 C API（如 gm_sm3_hash、gm_sm2_sign 等）。
  • 算法模块：sm2.c / sm3.c / sm4.c 分别实现椭圆曲线、杂凑与分组密码；SM2 侧依赖 sm2_z256 等域运算，与国标（如 GB/T 32918、SM3/SM4 标准）对齐。
  • 到浏览器：构建脚本将上述源码编译为单文件 gm.js，在线工具页仅负责输入校验与 UI，运算在编译后的模块中完成。

加密 / 解密在 C 层的典型流程

  • SM4（对称）：加密侧准备 128 bit 密钥与 IV（如 CBC）→ 对明文做 PKCS#7 填充 → 调用 gm_sm4_*_encrypt 得到密文；解密侧用同一密钥与 IV 调用 gm_sm4_*_decrypt → 去填充恢复明文。CTR/OFB/CFB 为流式加解密，无块填充概念，按模式更新计数器或链接变量即可。
  • SM2（公钥）：加密用对方 公钥 + 随机数 k → gm_sm2_encrypt 输出密文（椭圆曲线点 + 掩密文 + 校验）；解密用己方 私钥 → gm_sm2_decrypt 还原消息。签名 / 验签走 gm_sm2_sign / gm_sm2_verify，与加解密路径不同。
  • SM3（杂凑）：单向压缩，无「解密」。消息字节流 → gm_sm3_hash → 32 字节摘要；HMAC 在密钥与消息上再套一层 SM3 结构，由 gm_sm3_hmac 完成。

code 核心 C API 节选（gm.c）

以下为对外导出函数的一部分，展示从 C 层如何接到 SM3 与 SM2 密钥生成逻辑（完整实现见仓库源码）。

/* Emscripten / 本地共用入口：对外导出密码学算法 C API（SECMI / gm.c） */
GM_KEEPALIVE void gm_sm3_hash(const uint8_t *msg, size_t len, uint8_t out[32])
{
    sm3_digest(msg, len, out);
}

GM_KEEPALIVE int gm_sm2_generate_keypair(const uint8_t entropy[32], uint8_t priv[32], uint8_t pub[64])
{
    return sm2_generate_keypair(entropy, priv, pub);
}
